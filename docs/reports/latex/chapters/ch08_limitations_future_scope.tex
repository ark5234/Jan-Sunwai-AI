\chapter{Limitations and Future Scope}

No production-oriented civic platform is complete without explicit recognition of current boundaries. Jan-Sunwai AI delivers a strong operational baseline, but several limitations remain due to time, infrastructure, and deployment-context constraints.

\section{Current Limitations}

\subsection{Model and AI Limitations}

\begin{itemize}
\item Classification quality depends on local model behavior and image quality.
\item Ambiguous civic scenes may still require administrative triage despite hybrid logic.
\item Domain adaptation is limited to existing taxonomy and keyword/rule coverage.
\item Draft generation quality can vary by language complexity and scene context.
\end{itemize}

\subsection{Infrastructure Limitations}

\begin{itemize}
\item Throughput is constrained by local hardware, especially VRAM and CPU availability.
\item Single-node deployment is suitable for pilot and controlled workloads, not large multi-zone production scale without additional orchestration.
\item Current queue strategy is in-process and may require distributed message infrastructure for higher concurrency targets.
\end{itemize}

\subsection{Workflow and Data Limitations}

\begin{itemize}
\item Worker service-area quality depends on correctness and completeness of admin-maintained profiles.
\item Escalation semantics are configurable but not yet integrated with external authority APIs.
\item Public feed is intentionally anonymized, limiting deeper citizen-facing analytics.
\item Historical trend analytics can be expanded with richer time-series and SLA breach segmentation.
\end{itemize}

\subsection{Security and Governance Limitations}

\begin{itemize}
\item Security checks are strong at application layer but external perimeter controls (WAF, zero-trust segmentation) are deployment-dependent.
\item Email delivery remains stub-based until institutional SMTP integration is finalized.
\item Continuous dependency vulnerability tracking is not yet fully automated in release workflow.
\end{itemize}

\section{Future Scope}

\subsection{AI and Intelligence Enhancements}

\begin{enumerate}
\item Introduce domain fine-tuning or retrieval-augmented context for better local scene understanding.
\item Add multilingual voice-to-text intake for accessibility.
\item Integrate confidence calibration dashboards and model drift alerts.
\item Build image deduplication and duplicate complaint clustering using visual embeddings.
\end{enumerate}

\subsection{Operational Workflow Enhancements}

\begin{enumerate}
\item Implement SLA policy engine with configurable thresholds per department and issue class.
\item Add escalation automation with conditional approvals and supervisor override logs.
\item Extend assignment with route optimization and field workload balancing by travel cost.
\item Introduce richer citizen feedback loops and resolution quality scoring.
\end{enumerate}

\subsection{Platform and Deployment Enhancements}

\begin{enumerate}
\item Move from single-node compose to orchestrated deployments for higher availability.
\item Add asynchronous message broker for queue resilience and horizontal worker scaling.
\item Integrate centralized observability stack for metrics, traces, and security audit events.
\item Implement backup verification drills and disaster recovery runbooks with recovery point objectives.
\end{enumerate}

\subsection{Security and Compliance Enhancements}

\begin{enumerate}
\item Introduce periodic penetration testing workflow with report tracking.
\item Add secret rotation automation and hardware-backed key management integration.
\item Expand privacy controls for retention and redaction policies aligned with institutional mandates.
\item Include compliance templates for municipal procurement and audit reporting.
\end{enumerate}

\section{Adoption Considerations for Municipal Deployment}

\subsection{Institutional Readiness Factors}

Municipal deployment success is influenced by organizational readiness as much as software maturity. Key factors include role training quality, escalation policy clarity, and data-governance ownership. Even technically complete systems can underperform if operational owners do not have clear process responsibility.

\subsection{Change Management Considerations}

\begin{itemize}
\item Staff onboarding must include role-specific workflow rehearsals, not only UI tours.
\item Escalation policy language should be standardized before production launch.
\item Existing manual grievance channels should run in parallel during pilot transition.
\item Citizen communication templates should be localized and reviewed for clarity.
\end{itemize}

\subsection{Data Governance Considerations}

Future rollout should include an explicit policy for retention, redaction, access audits, and report publication limits. This is particularly important where complaint evidence may include personal or location-sensitive context.

\section{Research Continuation Opportunities}

The platform also creates opportunities for academic extension:

\begin{enumerate}
\item Comparative analysis of hybrid rule+LLM pipelines versus end-to-end multimodal classifiers in civic datasets.
\item Measurement of complaint resolution outcomes before and after AI-assisted intake.
\item Bias and fairness analysis across department categories and geographies.
\item Human-in-the-loop triage ergonomics and decision consistency studies.
\end{enumerate}

\section{Chapter Summary}

The current system is robust for pilot and structured deployment but remains intentionally conservative in distributed scaling and advanced automation. The future scope presented here provides a realistic pathway from project-grade implementation to institutional-grade civic infrastructure.
