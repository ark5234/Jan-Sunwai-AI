\chapter{Limitations and Future Scope}

Jan-Sunwai AI was built within a specific set of practical limits: host-machine VRAM, municipal infrastructure assumptions, and the security controls available to the organisation running it. This chapter records those limits so that the next stage of work can be planned against the conditions the system actually has, rather than against an idealised deployment environment.

\section{Current Limitations}

Three categories of limitation characterise the current release. The first concerns infrastructure capacity. The local-first deployment model binds AI response times to the available VRAM of the host machine. Under concurrent analysis requests, queue depth can grow in a way that produces perceptible response latency for citizens, particularly if the platform is operated at municipal scale on hardware below the recommended specification. The current synchronous portion of the inference pipeline is a secondary contributor to this constraint.

The second limitation concerns classification quality. Although the hybrid pipeline produces correct department assignments for the majority of clearly photographed civic issues, ambiguous scenes, poor lighting conditions, and non-representative image compositions continue to produce low-confidence results that are directed to the triage queue. The triage mechanism is a functional governance response to this limitation, but a high triage queue volume places an ongoing burden on administrative staff. Without a domain-specific fine-tuned model, this overhead is unlikely to be substantially reduced.

The third limitation concerns perimeter security. The application layer is hardened against the most common web vulnerabilities through the controls documented in Chapter 6. However, the external perimeter, including web application firewall configuration, network segmentation, intrusion detection, and zero-trust access controls, is entirely dependent on the infrastructure management practices of the host municipal organisation. Deployment without appropriate perimeter controls would expose the platform to network-level threats that the application layer alone cannot mitigate.

\section{Future Scope}

Three development priorities stand out as highest-value for a post-submission roadmap. The first is asynchronous pipeline scaling. Migration from the current single-host Compose topology to a distributed message-broker architecture would allow the AI inference component to be scaled independently of the API service, handling municipal-volume concurrency without degrading synchronous complaint operations. This migration would require queue infrastructure such as Redis or RabbitMQ and a corresponding change to the LLM queue worker design.

The second priority is localised model fine-tuning. The current zero-shot approach to vision classification produces usable results but would be substantially outperformed by a model fine-tuned on a labelled corpus of actual municipal complaint photographs annotated with correct department assignments. Assembling such a corpus from operational complaint records, after appropriate anonymisation, and using it to fine-tune a lightweight vision model would reduce misclassification rates and correspondingly reduce triage queue volume.

The third priority is automated SLA enforcement with external authority integration. The current escalation mechanism requires a department head or administrator to manually initiate escalation after recognising that a resolution deadline has passed. Automated SLA policy enforcement, where the system generates escalation events when configured time thresholds are breached, combined with direct API integration into higher-authority grievance systems such as CPGRAMS, would close the gap between institutional grievance response commitments and operational execution.

\section{Chapter Summary}

The limitations documented in this chapter are genuine constraints that must be addressed through engineering investment before the platform can be considered production-grade at full municipal scale. The future scope identifies the most impactful next steps towards that goal.
